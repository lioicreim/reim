## 기록 형식

### 날짜별 기록
```markdown
### YYYY-MM-DD

#### 카테고리 (수정/추가/개선/제거 등)
- **제목**: 간단한 설명
  - 상세 내용 1
  - 상세 내용 2
```

### 카테고리 설명
- **수정**: 버그 수정
- **추가**: 새로운 기능 추가
- **개선**: 기존 기능 개선
- **제거**: 기능 제거
- **리팩토링**: 코드 구조 개선
- **문서**: 문서화 작업

### 관리 팁
- 매일 작업 후 기록
- 중요한 결정이나 발견한 이슈도 기록
- 너무 상세하지 않게 핵심만 기록
- 주기적으로 정리 (월별로 섹션 분리)
